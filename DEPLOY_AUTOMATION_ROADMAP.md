# 🗺️ خارطة طريق: أتمتة نشر المشاريع الكبيرة (Full-Stack)

> الهدف النهائي: أن ينشر المستخدم مشروعه الكبير (مطعم، توصيل، أي شيء فيه
> backend + قاعدة بيانات) **بصفر خطوات يدوية**، تماماً كما تُنشر المواقع
> الثابتة اليوم.

---

## 1. أين نحن الآن

| نوع المشروع | الحالة | خطوات المستخدم |
|---|---|---|
| **موقع ثابت** (تعريفي/واجهة) | ✅ آلي بالكامل | زر نشر واحد → Vercel → رابط |
| **مشروع كبير** (backend + DB) | 🟡 نصف-آلي | 3 خطوات يدوية (أدناه) |

### الخطوات الثلاث اليدوية المتبقية للمشاريع الكبيرة
1. **إنشاء مستودع GitHub منفصل** يدوياً (مستودع فارغ خاص بالمشروع — **ليس** مستودع المنصّة).
2. **ربطه** في الداش (⋯ → GitHub) — الكود يدفع تلقائياً بعدها.
3. **زر Deploy to Render** + إدخال `MONGODB_URI` يدوياً في Render.

> 🛡️ ملاحظة أمان مثبّتة في الكود: المنصّة ترفض الآن ربط/دفع أي مشروع إلى
> مستودع المنصّة نفسه (`isPlatformRepo` في `backend/services/githubSync.js`)،
> لأن `git push --force` كان يمحو كود المنصّة. كل مشروع = مستودعه المنفصل.

---

## 2. الحقيقة الأساسية

**العائق أمام الأتمتة الكاملة ليس حجم الكود — بل مفاتيح وقرارات تحملها
المنصّة نيابة عن المستخدم.** بمجرد امتلاكها، تختفي كل الخطوات.

---

## 3. الجولتان

### 🟢 الجولة أ — أتمتة GitHub + Render (رخيصة، مطلوبة مبكراً)
تحوّل الخطوات 1–3 إلى **زر واحد → رابط `.onrender.com`**. تبقى قاعدة البيانات فقط يدوية.

- **إنشاء مستودع GitHub تلقائياً** لكل مشروع عبر GitHub API.
- **Render API الكامل**: إنشاء الخدمة + حقن متغيّرات البيئة + انتظار الرابط.

**تحتاج (مرة واحدة):**
- توكن GitHub بصلاحية `repo` (أو GitHub App).
- `RENDER_API_KEY`.

**متى:** لحظة **أول عميل غير تقني**، حتى قبل التسويق الجاد — لا يمكن مطالبة
صاحب مطعم بإنشاء مستودع GitHub؛ هذا وحده يُفشل البيع. والتكلفة تقريباً صفر.

### 🔴 الجولة ب — توفير قاعدة البيانات تلقائياً (المصروف المتكرّر)
تحقّق **صفر خطوات** — مثل المواقع الثابتة تماماً.

- بدل أن يُنشئ المستخدم MongoDB ويلصق الرابط، تُنشئ المنصّة **قاعدة منفصلة
  لكل مشروع داخل عنقود مُدار واحد** وتحقن بياناتها تلقائياً.
- الخيارات: MongoDB Atlas API / Render Postgres / Neon / عنقود مشترك.

**النموذج الاقتصادي (SaaS قياسي):**
> العميل يدفع لك اشتراكاً، وأنت تدفع للبنية التحتية. تكلفة القاعدة تُدرَج في
> السعر. لا تستأجر قاعدة لكل عميل — تشترك في **خدمة مُدارة واحدة** وتُقسّمها
> قاعدةً لكل مشروع، فالتكلفة تنمو تدريجياً مع نمو العملاء.

**متى:** مع بداية التوسّع التجاري الفعلي. يمكن البدء بعنقود مشترك رخيص ثم الترقية.

---

## 4. الخلاصة بجملة

> **استخدامك الشخصي** = استمر بالطريقة الحالية (لا داعي للأتمتة قبل وجود عملاء).
> **أول عميل حقيقي** = الجولة أ (رخيصة، شرط تجربة العميل).
> **التوسّع التجاري** = الجولة ب (تأجير/توفير قاعدة البيانات، المصروف المُدرَج في السعر).
