# 🤖 خارطة طريق: جولا بوت كمنتج مستقل (Chat Widget SaaS)

> الهدف النهائي: تحويل «جولا بوت» من **ميزة تنسخ ملفات داخل مشروع jaola** إلى
> **منتج مستقل يشترك فيه أي صاحب موقع** ويلصقه بسطر واحد — على غرار
> Intercom / Crisp / Tidio / Chatbase.

---

## 1. أين نحن الآن (مبنيّ ومدموج)

| اللبنة | الحالة | أين |
|---|---|---|
| ودجت مكتفٍ ذاتيّاً (زر عائم + لوحة دردشة) | ✅ | `backend/agents/jaolaBot.js` |
| White-label (يرث اسم/لون/رمز الموقع) | ✅ | `buildConfig` + `generateWidgetCSS` |
| قاعدة معرفة offline (قواعد + FAQ عربي + faq مخصّص) | ✅ | `defaultKB` + الودجت |
| ذكاء اصطناعي حيّ مع ارتداد آمن للقاعدة | ✅ | `POST /api/jaola-bot/chat` |
| توكن موقّع (HMAC) لهوية المشروع بلا تسريب مفاتيح | ✅ | `backend/agents/jaolaBotToken.js` |
| محدّد معدّل + سقف طول/مخرجات + CORS مخصّص | ✅ | `server.js` |
| إضافة بضغطة زرّ من لوحة التحكّم | ✅ | `Dashboard.jsx` (`handleAddBot`) |

**الحقيقة الأساسية:** ما بُني اليوم يغطّي **النواة التقنية** للمنتج. العائق
أمام «المنتج المستقل» ليس منطق البوت — بل **طريقة التوزيع والتخزين**: اليوم
نَنسخ الملفات داخل كل مشروع، وإعداداته محفورة داخل النسخة.

---

## 2. اللبنتان التاليتان (الأولوية — طلب المستخدم تسجيلهما)

### 2.1 المقتطف المُستضاف الواحد (Single Hosted Snippet)
بدل نسخ `jaola-bot.js` + `jaola-bot.css` في كل مشروع، يلصق صاحب **أي موقع**
سطراً واحداً:

```html
<script src="https://bot.jaola.com/embed.js?id=ABC123"></script>
```

- الكود يبقى على خادمنا → **تحديث واحد يحسّن كل البوتات فوراً**.
- يعمل على أي موقع (WordPress / Shopify / مخصّص)، لا مواقع jaola فقط.
- العميل لا يتعامل مع ملفات — سطر واحد.

**التنفيذ المقترح:** نقطة `GET /embed.js?id=…` تُقدّم نفس منطق الودجت الحالي
(نعيد استخدام `WIDGET_BODY`)، لكنها تحقن الإعداد المجلوب من سجلّ المستأجر بدل
الحفر الثابت. + نقطة `GET /api/jaola-bot/config?id=…` تُرجع إعداد المستأجر.

### 2.2 تعدّد المستأجرين (Multi-tenancy)
`id=ABC123` = مفتاح العميل. نحتاج **سجلّ مستأجرين** (Mongo collection) فيه لكل
عميل إعداده المستقل:

```
ABC123 → { ownerId, name, color, emoji, faq[], apiEnabled, quota, createdAt }
DEF456 → { … عميل مستقل تماماً }
```

- `embed.js?id=ABC123` يجلب إعداد هذا العميل وحده.
- خادم واحد يخدم آلاف البوتات المستقلّة، لكل منها لوحته وحصّته وفوترته.
- نقطة الدردشة الحيّة تُبدّل مصدر الهوية من التوكن الموقّع إلى `botId` من السجلّ
  (مع إبقاء التوكن الموقّع للمواقع المولّدة من jaola — توافق خلفي).

> 🛡️ ملاحظة أمان: مفاتيح المشاريع (MONGODB_URI…) تبقى مشفّرة ولا تُرسل للواجهة
> أبداً؛ نفس المبدأ يسري على أي سرّ في سجلّ المستأجرين.

## 3. لبنات لاحقة (بعد الأساس)

| اللبنة | الغاية |
|---|---|
| تخزين المحادثات + تحليلات | لوحة متابعة لكل عميل (أكثر الأسئلة، معدّل الحلّ) |
| ابتلاع المعرفة (رفع مستند/رابط → FAQ تلقائي) | تدريب البوت دون كتابة يدوية |
| تحويل لموظّف بشري (Inbox / Human handoff) | حين يعجز البوت |
| حصص وفوترة لكل عميل | ضبط تكلفة الذكاء + الطبقات (مجاني/Pro/Agency) |
| White-label للوكالات | إعادة بيع البوت باسم الوكيل |

---

## 4. نماذج التسويق المحتملة

1. **إضافة مدفوعة داخل jaola** (الأسهل): مجاني بالقواعد، ذكاء حيّ باشتراك شهري.
2. **SaaS مستقل «Jaola Bot»**: مقتطف واحد لأي موقع + لوحة إعداد + تسعير بالطبقات.
3. **إعادة بيع (Reseller/White-label)**: الوكالات ترخّصه وتبيعه لعملائها.

---

## 5. أول شريحة عمليّة موصى بها (عند البدء)

نقطة `GET /embed.js?id=…` تُقدّم الودجت + سجلّ مستأجرين بسيط (config لكل `id`)،
**دون لمس ما يعمل الآن** (النسخ الحالي داخل مشاريع jaola يبقى مساراً موازياً).
هذا يثبت النموذج من الطرف للطرف بأقل مخاطرة، ثم نبني عليه بقيّة اللبنات.
